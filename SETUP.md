# 🚀 ติดตั้ง Slip Bot บนเครื่องใหม่ (เครื่องสำรอง)

## ขั้นตอน "กดติดตั้งทีเดียว"
1. **ติดตั้ง Node.js 20+** (ครั้งเดียว) — https://nodejs.org
2. **คัดลอกทั้งโฟลเดอร์ `C:\ANto`** ไปเครื่องใหม่ (หรือ `git clone` แล้วคัดลอกไฟล์ลับเพิ่ม — ดูข้อ 3)
3. **คัดลอกไฟล์ลับจากเครื่องหลัก** มาวางในโฟลเดอร์ (ไฟล์เหล่านี้ไม่ได้อยู่ใน git):
   - `.env` (ตั้งค่า + คีย์ทั้งหมด) — หรือก๊อป `.env.example` เป็น `.env` แล้วเติมเอง
   - `credentials.json` (Google OAuth client)
   - `tokens/google_token.json` (Google token ที่ authorize แล้ว)
   - *(ไม่บังคับ)* `data/slip_jobs.json`, `data/slipbot.db` — ถ้าอยากได้ประวัติงาน/กันซ้ำต่อเนื่อง
4. **ดับเบิลคลิก `install.bat`** → ติดตั้ง deps + PM2 + เริ่มบอทให้อัตโนมัติ

## คำสั่งที่ใช้บ่อย
```
npx pm2 status            # ดูสถานะ
npx pm2 logs slip-bot     # ดู log
npx pm2 restart slip-bot  # รีสตาร์ท
npx pm2 stop slip-bot     # หยุด
```
แดชบอร์ด: http://localhost:3000

## ⚠️ ข้อควรระวัง (สำคัญ)
- **ห้ามรันบอท 2 เครื่องพร้อมกันด้วย Telegram token เดียวกัน** — Telegram ยอมให้ดึงข้อความ (poll) ได้ทีละเครื่อง ถ้าซ้อนจะขึ้น error 409 Conflict
- เครื่องสำรองให้**เปิดเฉพาะตอนเครื่องหลักดับ** (failover แบบสลับมือ): หยุดบอทเครื่องหลักก่อน → เริ่มเครื่องสำรอง
- ข้อมูลจริงอยู่บน Google Sheets อยู่แล้ว → เครื่องสำรองเชื่อม Sheets เดิม เห็นข้อมูลครบทันที

## สตาร์ทอัตโนมัติหลังรีบูต (ไม่บังคับ)
```
npx pm2 startup     # ครั้งเดียว (ต้องรันใน PowerShell แบบ Admin)
npx pm2 save        # บันทึกรายการโปรเซสปัจจุบัน
```
